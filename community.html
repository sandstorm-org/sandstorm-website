Get involved in the global Sandstorm community

Join app authors and many thousands of users

Welcome!

We're so glad you're interested in getting involved! We're a movement of app makers, users, open source enthusiasts, self-hosting fanatics, enterprise IT staffers, documentation writers, and more, working together on making server apps safe and easy for everyone to use. Here's how you can contribute.

Connect with us

  • Fediverse
  • GitHub
  • Zulip Chat
  • Mailing List
  • YouTube

Help others

  • Use & Share Sandstorm

    Self-host Sandstorm to install apps from the App Market. Then share by email or by sharing link to invite friends and colleagues to collaborate.

  • Give a Talk

    Get started with our speaker kit to prepare your presentation. Send us an email if you need help with finding a venue or preparing for your talk.

  • Discuss & Answer Questions

    User help happens on the mailing list; join us and contribute! Real-time chat happens on Zulip chat. You can see old IRC chat archives or Matrix too.

  • Write a Blog Post

    Share your Sandstorm experiences with the world by writing a blog post. Send us a link and we'll share it with the community.

Help the project

  • Package an App

    Check out our tutorial to get started packaging apps for the App Market. If you get stuck, email the dev list for help or hop in our Zulip chat.

  • Provide Feedback

    To provide suggestions for each app: visit the Sandstorm App Market, click on the app, click on the app's Code link, then navigate to its issue tracker. You can report bugs in Sandstorm itself on our issue tracker.

  • Design App Icons

    Many apps packaged for Sandstorm need icons and design help. If you have design skills and would love to help improve apps, send us an e-mail and we'll connect you with apps in need.

  • Usability Test

    Usability tests are a great way to identify user problems and areas for improvement. Find volunteers, record them as they try Sandstorm, review the videos, then file issues. For more help read Usability 101 by Jakob Nielsen.

  • Improve the Documentation

    The Sandstorm documentation lives in the main GitHub repository for Sandstorm. Listen to the Zulip chat and mailing list, identify a common question, then submit a pull request.

  • Contribute to the Core

    Sandstorm's code is open source and lives on GitHub. To contribute bug reports, patches, design ideas, documentation, pull requests, and more check out our guide to contributing.

  • We'd love to hear more ideas
